OpenAI SDK Compatible APIs: Full List 2026

Published Jun 1, 2026· Updated Jun 1, 2026· 7 min read· By BurnCloud

If your code already uses the OpenAI SDK, you can reach dozens of models — Claude, Gemini, DeepSeek, Qwen and more — by changing just two lines. This guide explains what “OpenAI SDK compatible” really means, shows the one-time switch, lists every model reachable this way through BurnCloud in 2026, and covers what carries over (streaming, tools, frameworks) and what doesn't.

What “OpenAI SDK compatible” means

An OpenAI-compatible API accepts the exact same request shape and returns the exact same response shape as OpenAI's /v1/chat/completions endpoint. So the official openai library — and everything built on it — works unchanged once you repoint it.

  • Same endpoints and request/response JSON
  • Drop-in: change base_url and api_key, nothing else
  • Pick any model by its model string
  • Streaming, tools and JSON mode use identical syntax

The two-line switch

  1. Set base_url to https://api.burncloud.com/v1.
  2. Set api_key to your BurnCloud key.
  3. Choose any supported model by its model string.
  4. Run your existing code — unchanged.

That's the whole migration. Here it is in the OpenAI SDK:

import openai

client = openai.OpenAI(
    base_url="https://api.burncloud.com/v1",
    api_key="your-burncloud-key"
)

response = client.chat.completions.create(
    model="deepseek-v4",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Explain how APIs work"}
    ],
    max_tokens=1000
)

print(response.choices[0].message.content)
import OpenAI from "openai";

const client = new OpenAI({
  baseURL: "https://api.burncloud.com/v1",
  apiKey: "your-burncloud-key",
});

const response = await client.chat.completions.create({
  model: "deepseek-v4",
  messages: [
    { role: "user", content: "Explain how APIs work" }
  ],
});

console.log(response.choices[0].message.content);
curl https://api.burncloud.com/v1/chat/completions \

Streaming and tools work unchanged

Because the wire format matches OpenAI's, the two patterns you rely on most — streaming and function calling — are identical no matter which model string you use.

Streaming responses

for chunk in client.chat.completions.create(
    model="deepseek-v4",
    messages=[{"role": "user", "content": "Hello"}],
    stream=True
):
    print(chunk.choices[0].delta.content or "", end="")

Function calling

tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "parameters": {
            "type": "object",
            "properties": {
                "location": {"type": "string"}
            }
        }
    }
}]

Frequently asked questions

What does OpenAI SDK compatible mean?+
It means the API accepts the same request and response shape as OpenAI's, so the official openai SDK works against it after you change the base_url and key — no rewrite required.
Which frameworks work with BurnCloud?+
Anything built on the OpenAI SDK, including LangChain, LlamaIndex, LiteLLM and most agent frameworks. Point them at the BurnCloud base_url. See the quickstart →
Do I have to change my code?+
Only two lines: the base_url and the api_key. After that you select any supported model by its model string.
Does function calling work?+
Yes. Tool/function calling, streaming and JSON mode use the same OpenAI syntax across supported models.
Can I use Claude and GPT through the OpenAI SDK?+
Yes. Through BurnCloud, Claude, Gemini, DeepSeek, Qwen and others are all reachable via the OpenAI SDK by changing only the model string.
