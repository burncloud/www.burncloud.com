How we verify suppliers

  1. Submit official API account proof
    Every supplier provides evidence of a genuine, first-party provider account.
  2. BurnCloud verifies authenticity
    We confirm the account is real and directly provisioned by the official provider.
  3. Regular sampling tests
    Live requests are sampled and checked to confirm the served model matches what was requested.
  4. Automatic removal on failure
    Any supplier that fails a sampling test is pulled from rotation immediately.

Frequently asked questions

Does BurnCloud log my prompts or responses?+
No. The gateway stores no prompts, responses, usage patterns, or IP addresses. Because the code is open source, you can search it for any logging or storage path and confirm there is none.
How can I verify what is running?+
Call verify.burncloud.com to read the live version, commit, and build hash, then build the same commit yourself and compare the sha256 hash against what's deployed.
Can I self-host BurnCloud?+
Yes. BurnCloud is a single Rust binary under an MIT license. Clone the repo, build it, and run it on your own infrastructure for full control.
How are suppliers verified?+
Every supplier submits proof of an official API account, which BurnCloud verifies for authenticity, then runs regular sampling tests and removes any supplier that fails.
Is BurnCloud compliant for enterprise use?+
BurnCloud Ltd. is UK registered and GDPR compliant. A data processing agreement is available, and a SOC2 report is available on request.
